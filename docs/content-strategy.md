# Content Strategy

## Brand

Long-term series brand: **AI in Action**.

Cursor may appear as one tool in the workflow. The series should not be permanently named after a single vendor.

Owner voice: Senior Technology Architect — practical engineering, not hype.

Audience: engineers, architects, hiring managers, peers.

## Content package per application

Every Check app should eventually ship with:

1. Live application (free educational demo)  
2. Public (or curated) GitHub repository  
3. Long-form article (LinkedIn + hub mirror)  
4. Short promotional / explainer video  
5. Architecture diagram  
6. Product journey diagram  
7. Lessons-learned post  
8. Feature-feedback / roadmap update post  
9. Optional technical deep dive  

## Cadence

Aim for about **two LinkedIn posts per week** when sustainable. **Quality wins** over volume.

## Story arc template (every major article)

```
Problem → Product Thinking → Requirements → Architecture
→ AI-Assisted Development → Testing → CI/CD → Cloud
→ Feedback → Continuous Improvement
```

Always answer: **What engineering lesson did building this teach?**

9. **What comes next** — polish, feedback, optional auth later

# 90-Day Plan

**Horizon:** ~through September–early October 2026  
**Goal:** Enter Phase 3 optional identity/persistence carefully; prepare Phase 4 advanced capabilities as engineering learning — still educational showcase framing.

## Outcomes

1. Managed identity provider selected (decision logged) — if pursuing accounts  
2. Optional registered accounts available on at least one app (or clear timeline with privacy docs complete)  
3. Cloud history / sync path defined and partially implemented  
4. Privacy policy, terms, retention, and deletion model ready for accounts  
5. Centralized capability policy design in place (advanced features may still be empty)  
6. Feature flag approach chosen  
7. Testing / observability improvements underway  

## Workstreams

### Identity and persistence

- [ ] Complete IdP evaluation against [../docs/authentication-strategy.md](../docs/authentication-strategy.md)  
- [ ] Implement optional registered users on pilot app  
- [ ] Cloud persistence for settings/history  
- [ ] Local-first path remains usable  

### Policy and privacy

- [ ] Publish privacy/terms appropriate to stored data  
- [ ] Retention + deletion flows  
- [ ] Capability policy design (even if in-process module)  

### Advanced capabilities (architecture / engineering learning)

- [ ] Feature flags for sensitive capabilities  
- [ ] Map [../docs/capability-maturity.md](../docs/capability-maturity.md) to capability keys  
- [ ] Accessibility and automated testing hardening  

## Explicit non-goals (unless learning goals clearly require otherwise)

- Full cross-app dashboard  
- Wearable integrations in production  
- Health-regulatory certification  
- Private commercial strategy documentation in this repo  

## Exit criteria

Phase 3 checklist substantially underway; Phase 4 items designed; [../docs/decision-log.md](../docs/decision-log.md) contains IdP decision if auth is pursued; quarter success does not require a commercial launch.

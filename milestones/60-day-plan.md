# 60-Day Plan

**Horizon:** ~through August 2026  
**Goal:** Finish Phase 1 credibility work; begin Phase 2 reusable foundations without overengineering.

## Assumes 30-day outcomes

SleepCheck polish underway or largely done; hub article published; LinkedIn #2 optional; roadmap repo public and linked from hub/apps.

## Outcomes

1. AI in Action #2 LinkedIn presence improved (and/or optional video)  
2. Application READMEs and architecture docs at “recruiter-ready” quality  
3. Shared design/system conventions documented (tokens, layout patterns)  
4. Shared user model described (schema-level), not necessarily implemented  
5. Telemetry / observability and deployment conventions drafted  
6. Local history / sessions shipped or in PR for at least one app  
7. Authentication requirements written; IdP shortlist scored  

## Workstreams

### Phase 1 closeout

- [ ] SleepCheck LinkedIn / optional video  
- [ ] Close critical polish bugs  
- [ ] Summarize feedback themes in decision log or product notes  

### Phase 2 foundations

- [ ] Inventory duplicated UI/theme utilities across RetireCheck and SleepCheck  
- [ ] Decide extract-now vs document-later for each item  
- [ ] Draft shared user model fields (identity-agnostic)  
- [ ] Document deployment patterns (Vercel/Render/etc. as applicable)  

### Toward optional identity (prep only / early spikes)

- [ ] Privacy and retention draft outline  
- [ ] IdP evaluation spike (time-boxed)  
- [ ] Capability naming conventions drafted from maturity matrix  

## Explicit non-goals (60 days)

- Multi-app SSO in production  
- Private commercial systems implementation  
- Monorepo migration unless clearly justified  

## Exit criteria

Phase 2 checklist started with concrete shared artifacts; auth still not required in production unless a deliberate early spike succeeded and privacy docs exist.
